Prompt、Skill、Agentは似た言葉に見えますが、実務では役割が違います。違いを曖昧にしたままAIを使うと、毎回その場の会話になり、成果物が残りません。

AI業務改善では、まずこの3つを分けて考えます。

- Prompt = その場の指示
- Skill = 再利用可能な作業手順
- Agent = 判断しながら実行する担当者

この区別ができると、ChatGPT、Codex、Claude Code、Cursor、Gemini、Obsidianを役割で組み合わせやすくなります。

## Promptはその場の指示

Promptは、AIに渡す指示です。最も小さな単位で、すぐに試せます。

ただし、実務で使うPromptには最低限の構造が必要です。

- 目的
- 前提
- 入力
- 制約
- 出力形式
- 確認観点

例えば「問い合わせ文を作って」ではなく、「AI業務改善に関心がある中小企業向けに、過度な営業感を出さず、初回相談前に整理する項目を含む問い合わせ文を作って」と書くと、出力の品質が安定します。

Promptは試行錯誤の入口です。うまくいったPromptは、次にSkillへ昇格させます。

## Skillは再利用可能な作業手順

Skillは、繰り返し使う作業手順です。Promptよりも一段上の単位で、作業の流れ、禁止事項、検証方法を含みます。

Skillには次のような情報を入れます。

- どんなときに使うか
- 最初に読むファイル
- 変更してよい範囲
- 変更してはいけない範囲
- 作業手順
- 検証コマンド
- 完了条件

このサイトの作業で言えば、「Astro記事追加Skill」「公開前チェックSkill」「OGP画像差し替えSkill」のように分けられます。

Skillの価値は、次回の作業を短くすることです。毎回説明しなくても、同じ品質で作業を始められる状態を目指します。

## Agentは判断しながら実行する担当者

Agentは、目的を受け取り、ツールを使い、判断しながら作業する担当者です。

例えばCodexは、リポジトリを読み、ファイルを編集し、`npm run build` を実行し、git commitまで進められます。これは単なるPromptではなく、作業者としての振る舞いです。

ただし、Agentには必ず権限境界が必要です。

- どのディレクトリを触ってよいか
- どのコマンドを実行してよいか
- どの情報を読んではいけないか
- どこから人間の承認が必要か
- 何をもって完了とするか

Agentは強力ですが、権限を広げすぎると危険です。人間が判断すべき作業は残します。

## MCPは外部能力をつなぐ仕組み

MCPは、AIが外部ツールやデータにアクセスするための接続口として考えると分かりやすいです。

例えば、ファイル、ブラウザ、GitHub、メール、データベース、社内ツールなどをAIが扱えるようにする仕組みです。

便利ですが、接続できる範囲が広がるほどリスクも増えます。MCPを使う場合は、何に接続するのか、何を読めるのか、何を書き込めるのかを明確にします。

このサイトでは、秘密情報や本番サーバー操作を扱わない方針を置いています。

## ToolはAgentが使う道具

Toolは、Agentが実際に使う道具です。ファイルを読む、検索する、buildする、画像を作る、ブラウザで確認するなどの操作がToolにあたります。

Toolを使わせるときは、目的と制約をセットにします。

悪い例:

```text
必要なら何でもやって
```

良い例:

```text
src/content/articles 配下に記事を追加し、npm run build で確認する。本番サーバーには接続しない。
```

Toolは作業を進める力です。だからこそ、使う範囲を絞る必要があります。

## Hookは特定のタイミングで動く処理

Hookは、特定のタイミングで動く処理です。例えば、commit前にlintを実行する、build後にファイル一覧を確認する、記事追加時にfrontmatterを検査する、といった使い方があります。

Hookを入れると、確認漏れを減らせます。ただし、最初から複雑にしすぎる必要はありません。

公開直後のサイトでは、まず人間がチェックリストで確認し、繰り返しが見えてきたらHook化するのが現実的です。

## Human Approvalは人間の承認

Human Approvalは、人間の承認が必要な地点です。

このサイトの運用では、次の作業は人間が判断します。

- 本番サーバーへのアップロード
- お名前.com管理画面操作
- DNS変更
- Google Search Console操作
- 顧客情報や個人情報の扱い
- 公開前の最終判断

AIに作業を任せるほど、人間の承認ポイントは明確にしておく必要があります。

## 実務での使い分け

最初からAgentを作ろうとしないほうが安定します。

おすすめの順番は次の通りです。

1. Promptで一度試す
2. うまくいったらSkillとして保存する
3. 入力、制約、完了条件が固まったらAgentに渡す
4. 危険な作業にはHuman Approvalを入れる
5. 評価観点をEvalとして残す

この順番なら、AI活用が属人的なコツではなく、再利用できる業務改善になります。
