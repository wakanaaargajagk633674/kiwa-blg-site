Codexに静的サイト制作を任せるときは、最初に「どこまで触ってよいか」と「何で完了とするか」を決める。

## 作業単位

初期構築では、以下の単位に分けるとレビューしやすい。

- 依存関係と設定
- レイアウトとグローバルCSS
- 共通コンポーネント
- ページ追加
- ビルド検証

## 完了条件

`npm run build` が通ること、作成したページが静的HTMLとして出力されること、不要なサーバー処理を追加していないことを確認する。

> AIに任せる範囲を広げるほど、検証条件は具体的にする。

次回以降は、記事詳細、OGP画像、サイトマップ、問い合わせ導線の順に広げる。
